# 动画文件准备指南

> 本文档说明如何为 VRM 角色准备和配置外部动画文件，使角色能够做出丰富的肢体动作。

---

## 一、当前动画系统架构

```
playMotion("wave")
       │
       ▼
 ┌─────────────┐     有外部文件？    ┌──────────────────┐
 │  MOTION_MAP  │ ──── 是 ────────→ │ AnimationController │ → GLB/BVH 骨骼动画
 │  语义名映射   │                   │  加载+缓存+播放     │
 └─────────────┘     否             └──────────────────┘
       │
       ▼
 ┌─────────────────┐
 │ 程序化动画（兜底） │ → 11种硬编码骨骼动画 / 6种表情
 └─────────────────┘
```

**外部动画文件优先级高于程序化动画。** 放了文件就用文件，没放就用代码里写死的预设。

## 二、需要什么文件

### 2.1 支持的格式

| 格式 | 后缀 | 说明 | 推荐度 |
|------|------|------|:---:|
| **GLB** | `.glb` | GLTF 二进制，包含骨骼+动画数据 | ⭐⭐⭐ |
| **BVH** | `.bvh` | 动作捕捉文本格式，纯骨骼旋转数据 | ⭐⭐ |
| **JSON** | `.json` | 自定义格式，手工制作或脚本导出 | ⭐ |

**推荐使用 GLB 格式**——兼容性最好，Blender/Mixamo 都能直接导出。

## 三、文件从哪来

### 方法一：从 Mixamo 下载（最简单）

**网站**：https://www.mixamo.com（需 Adobe 账号，免费）

**步骤**：

1. 登录后，左侧角色列表任选一个（只用于预览，不影响最终效果）
2. 右侧搜索框输入动作名（如 `idle`、`walk`、`wave`）
3. 点击动画预览，满意后点右侧 `Download` 按钮
4. 下载设置：
   - **Format**: `FBX`（后续用 Blender 转 GLB）
   - **Skin**: `Without Skin`（只要骨骼动画，不要模型）
   - **Keyframe Reduction**: `none`（保留所有关键帧）
5. 下载得到 `.fbx` 文件

**FBX 转 GLB**（需要用 Blender）：

1. 打开 Blender
2. `File` → `Import` → `FBX (.fbx)` → 选择下载的文件
3. `File` → `Export` → `glTF 2.0 (.glb/.gltf)`
4. 格式选 `glTF Binary (.glb)`
5. 勾选 `Export: Selected Objects`（如果有多个对象）
6. 导出，得到 `.glb` 文件

### 方法二：从其他资源网站下载

| 网站 | 格式 | 说明 |
|------|------|------|
| https://www.mixamo.com | FBX | Adobe 免费，动画质量高 |
| https://readyplayer.me/mixamo | FBX | Mixamo 镜像 |
| https://sketchfab.com | GLB/FBX | 部分免费，需搜索带动画的模型 |
| https://www.bvhacker.com | BVH | BVH 编辑/转换工具 |

### 方法三：自己用 Blender 制作

1. 在 Blender 中创建骨骼动画
2. `File` → `Export` → `glTF 2.0 (.glb)`
3. 导出时确保勾选 `Animation` 选项

**所有角色共享同一套动画文件目录。** 系统会根据骨骼名自动重定向，不同 VRM 模型可以共用同一套动画。

## 五、如何配置

在角色的 `profile.json` 中添加 `animations` 字段：

```json
{
  "characterId": "hatsune_miku",
  "name": "初音未来",
  "appearance": {
    "modelPath": "assets/model/miku.vrm",
    "styleType": "vrm",
    "animations": {
      "idle": "idle.glb",
      "walk": "walk.glb",
      "wave": "wave.glb",
      "sit": "sit.glb",
      "run": "run.glb",
      "happy": "happy.glb",
      "dance": "dance.glb"
    }
  }
}
```

**键名**（如 `"idle"`）是语义名，对应 `playMotion("idle")` 时传入的名称。
**值**（如 `"idle.glb"`）是 `static/animations/` 目录下的文件名。

## 六、骨骼名映射说明

Mixamo 导出的 FBX 中，骨骼名带有 `mixamorig:` 前缀，如：

```
mixamorig:Hips
mixamorig:Spine
mixamorig:Spine1
mixamorig:Head
mixamorig:LeftUpperArm
...
```

系统内置了完整的映射表，会自动转换为 VRM 标准骨骼名：

```
mixamorig:Hips         → hips
mixamorig:Spine        → spine
mixamorig:Spine1       → chest
mixamorig:Head         → head
mixamorig:LeftUpperArm → leftUpperArm
...
```

**无需手动处理骨骼名，Mixamo 的 FBX 直接转 GLB 就能用。**

如果使用非 Mixamo 来源的动画，骨骼名需要符合 VRM 标准（全小写驼峰命名）。完整映射表见 `static/js/animation_controller.js` 中的 `BONE_NAME_MAP`。

## 七、验证动画是否生效

1. 将 `.glb` 文件放入 `static/animations/`
2. 在 `profile.json` 中配置 `animations`
3. 启动程序，切换到该角色
4. 打开浏览器开发者工具（F12），查看控制台输出：
   ```
   [AnimCtrl] Loading GLB: static/animations/wave.glb
   [AnimCtrl] GLB loaded: wave duration: 2.50s
   [Animation] VRM external anim: wave
   ```
5. 如果看到 `external anim` 说明外部动画生效
6. 如果看到 `VRM skeletal` 说明回退到了程序化动画（文件未找到或加载失败）

## 八、常见问题

### Q: 放了文件但没生效？

检查以下几点：
- 文件是否在 `static/animations/` 目录下（不是子目录）
- `profile.json` 中的文件名是否正确（大小写敏感）
- `.glb` 文件是否包含动画数据（用 Blender 打开确认）
- 控制台是否有错误信息

### Q: 动作幅度太大/太小？

GLB 中的动画数据是相对于原始角色体型的。如果角色体型差异大，可能需要在 Blender 中调整动画的骨骼缩放。

### Q: 能用 .fbx 文件吗？

目前不支持直接加载 `.fbx`。需要先用 Blender 转换为 `.glb`。

### Q: 不同角色能用不同的动画吗？

可以。每个角色的 `profile.json` 独立配置 `animations`，指向不同的文件。

### Q: 不配置 animations 会怎样？

完全正常运行。系统会使用内置的 11 种程序化骨骼动画 + 6 种表情，只是动作种类较少。
