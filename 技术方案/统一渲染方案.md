# 统一渲染方案实现计划

## 一、目标

将当前"每角色独立 HTML"的渲染方式，重构为"统一渲染页面 + 模型动态加载"方式。

**预期收益**：
- 修复 THREE_VRM 命名 bug 后验证统一渲染可行性
- SDK 只加载一次，切换角色不重新加载
- 模型 LRU 缓存，重复切换从 ~500ms 降至 ~50ms
- 错误状态回传 Python，降级 UI 更友好

---

## 二、风险控制策略

采用**渐进式迁移**，保留旧模式作为降级方案：

```
renderer.html 可用？
    ├── 是 → 使用统一渲染（新模式）
    └── 否 → 降级到独立 HTML（旧模式）
```

### 阶段 4：优化和清理 ✅

**目标**：优化内存管理，清理旧代码

- [x] 动态缓存大小（基于内存压力）

## 六、回退方案

如果统一渲染方案出现无法解决的问题：

1. **保留旧模式**：`model_widget.py` 中的 `_load_legacy_mode()` 方法
2. **自动降级**：renderer.html 加载失败时自动切换
3. **手动切换**：配置项控制使用哪种模式
