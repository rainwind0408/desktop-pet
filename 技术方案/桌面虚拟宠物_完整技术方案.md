# 桌面虚拟宠物项目技术方案文档

---

## 一、项目概述

本项目旨在开发一个具备AI陪伴功能的桌面虚拟宠物应用，核心特色包括：

- 多角色形象系统，支持Live2D高级交互效果
- 每个角色拥有独立的性格设定和记忆库
- 前端可实时修改角色形象和性格
- 自动化的角色切换和数据更新机制
- 时空环境感知能力，宠物可感知时间、天气、纪念日等
- 多模态感知能力，宠物可感知用户正在收听的音乐和观看的视频，并主动交互

## 四、后端核心逻辑实现

### 4.1 角色管理器设计

```python
import json
import os
from typing import Dict, Optional
from langchain.chains import ConversationChain
from langchain.memory import VectorStoreRetrieverMemory
from langchain_community.vectorstores import FAISS
from langchain_community.docstore.in_memory import InMemoryDocstore
import faiss

class CharacterManager:
    def __init__(self):
        self.current_character_id: Optional[str] = None
        self.llm_chain: Optional[ConversationChain] = None
        self.memory: Optional[VectorStoreRetrieverMemory] = None
        self.characters_dir = "characters"

    def get_available_characters(self) -> list:
        """获取所有可用角色列表"""
        if not os.path.exists(self.characters_dir):
            return []
        return [d for d in os.listdir(self.characters_dir)
                if os.path.isdir(os.path.join(self.characters_dir, d))]

    def load_character_profile(self, character_id: str) -> Dict:
        """加载角色档案"""
        profile_path = os.path.join(self.characters_dir, character_id, "profile.json")
        if not os.path.exists(profile_path):
            raise FileNotFoundError(f"角色档案不存在: {profile_path}")
        with open(profile_path, 'r', encoding='utf-8') as f:
            return json.load(f)

    def load_character_memory(self, character_id: str):
        """加载角色专属记忆库"""
        memory_path = os.path.join(self.characters_dir, character_id, "memory.faiss")
        # 创建FAISS索引
        dimension = 1536  # 向量维度
        index = faiss.IndexFlatL2(dimension)
        vectorstore = FAISS(
            embedding_function=None,  # 需要实现具体的embedding
            index=index,
            docstore=InMemoryDocstore(),
            index_to_docstore_id={}
        )
        # 如果存在现有记忆库，加载它
        if os.path.exists(memory_path):
            vectorstore.load_local(memory_path)
        return VectorStoreRetrieverMemory(retriever=vectorstore.as_retriever())

    def switch_character(self, character_id: str) -> Dict:
        """切换角色"""
        # 1. 加载角色档案
        profile = self.load_character_profile(character_id)
        # 2. 加载角色记忆库
        self.memory = self.load_character_memory(character_id)
        # 3. 创建AI对话链
        self.llm_chain = ConversationChain(
            llm=your_llm_model,  # 需要配置具体的LLM
            memory=self.memory,
            verbose=True
        )
        # 4. 更新当前角色ID
        self.current_character_id = character_id
        print(f"成功切换至角色：{profile['name']}")
        return profile

    def chat_with_character(self, user_input: str) -> str:
        """与当前角色对话"""
        if not self.llm_chain:
            return "请先选择一个角色！"
        return self.llm_chain.run(user_input)

    def save_character_profile(self, character_id: str, profile: Dict):
        """保存角色档案"""
        profile_path = os.path.join(self.characters_dir, character_id, "profile.json")
        # 确保目录存在
        os.makedirs(os.path.dirname(profile_path), exist_ok=True)
        with open(profile_path, 'w', encoding='utf-8') as f:
            json.dump(profile, f, ensure_ascii=False, indent=2)
```

## 五、时空环境感知模块

### 5.1 打造宠物的"时空感官中枢"

在后端编写一个轻量级的环境感知模块，负责实时采集时间、纪念日进度以及外部天气数据，并将其转化为 AI 能够理解的文本描述。

#### 时间与纪念日计算

利用 Python 的 `datetime` 库获取当前绝对时间，并结合角色档案中的 `first_met_timestamp` 动态计算出相识天数、是否临近生日等相对时间信息。

#### 季节与温度感知（接入真实天气 API）

为了让宠物知道"今天下雨了"或者"最近降温了"，需要接入第三方的天气 API（如和风天气、OpenWeatherMap 等）。

- **实现逻辑**：在程序启动时，或每天凌晨定时触发一次网络请求。根据用户设置的地理位置（或在初次启动时通过 IP 自动定位），拉取当天的天气状况（晴/雨/雪）、实时气温、昼夜温差等信息。
- **本地缓存**：为了避免频繁调用 API 产生延迟或消耗额度，可以将当天的天气数据缓存在本地的 `environment_cache.json` 中，每隔几小时或跨天时再重新刷新。

### 5.2 动态组装"环境提示词"

在每一次向宠物发送消息之前，Python 后端不仅要传递聊天内容，还要在后台悄悄组装一段包含当下时空信息的系统提示词。

#### Prompt 模板设计示例

```text
[当前环境感知]
- 物理时间：2026年5月17日，星期日，上午11点（初夏时节）。
- 天气状况：江苏南通，多云转晴，当前气温 24℃，微风，体感舒适。
- 情感羁绊：今天是你们相识的第 100 天纪念日。

[行为指导]
你是主人的专属伴侣。请基于上述环境信息进行回复。例如，如果外面下雨，可以提醒主人带伞；
如果是纪念日，可以在对话中自然地流露出开心和期待；如果现在是深夜，说话语气要轻柔一些，
催促主人休息。不要生硬地报出数据，要把这些信息融入到日常的关心或闲聊中。
```

#### 代码实现逻辑

每次调用 LangChain 或大模型 API 前，先运行环境感知模块，用 f-string 格式化上述模板，将其作为 `system_message` 或对话的前缀拼接到用户的输入中，一并发送给大模型。

### 5.3 情绪权重的动态调节

除了直接在 Prompt 里写明信息，还可以将这些环境因素转化为量化的"情绪参数"，进一步影响 AI 的语气。

#### 温度与情绪挂钩

设定简单的规则引擎。例如，当 API 返回气温低于 10℃ 时，在后端标记 `mood_tag = "concern_cold"`（寒冷关怀模式）。将这个标签传给前端，前端可以控制 Live2D 模型做出"抱紧自己"、"哈气暖手"的动作，同时 Prompt 也会指示 AI 使用更温暖、黏人的词汇。

#### 特殊日子的兴奋度

当检测到是第 100 天、情人节或用户的生日时，将 `excitement_level`（兴奋等级）调高。AI 在生成回复时，会倾向于使用更多的感叹号、颜文字，甚至主动开启关于"回忆过去"的话题。

### 5.4 前后端的视觉联动

为了让这种感知更加立体，后端在注入 Prompt 的同时，可以将关键的环境标签同步给前端界面。

#### 场景化背景切换

如果后端感知到今天外面在下大雨，可以通过接口告诉前端：`current_weather = rain`。前端接收到信号后，自动在宠物身后的透明窗口叠加一层逼真的雨滴特效，或者让宠物撑起一把虚拟的小伞。

#### 服饰自动适配

结合换装系统，当气温骤降时，后端甚至可以建议或直接控制前端为宠物换上"冬季限定"的厚衣服或围巾，并让 AI 在对话中说："外面好冷呀，我把小围巾都戴好啦，主人也要多穿点哦！"

## 六、多模态感知方案

### 6.1 概述

本方案为AI桌宠提供多模态感知能力，使其能够感知用户当前正在观看的视频和收听的音乐，并通过智能分析实现主动交互。方案采用"接口直连+低频分析"的优化架构，有效控制资源消耗。

### 6.2 系统架构

```
+-------------------+     +-------------------+     +-------------------+
|  音乐感知模块     |     |  视频感知模块     |     |  交互决策模块     |
|  - 音乐软件接口   |     |  - 视频软件接口   |     |  - 情境分析       |
|  - 历史数据分析   |     |  - 智能抽帧分析   |     |  - 主动交互       |
+-------------------+     +-------------------+     +-------------------+
        |                       |                       |
        v                       v                       v
+---------------------------------------------------------------+
|                       数据采集与处理层                          |
|  - 系统级接口调用   - API数据获取   - 屏幕截图分析             |
+---------------------------------------------------------------+
        |
        v
+---------------------------------------------------------------+
|                        基础设施层                               |
|  - 操作系统接口   - 网络通信   - 数据存储   - AI模型服务      |
+---------------------------------------------------------------+
```

### 6.3 音乐感知方案

#### 6.3.1 实时播放感知

**系统级接口方案：**

| 平台 | 技术方案 | 支持应用 | 获取数据 |
|------|----------|----------|----------|
| Windows | `SystemMediaTransportControls` (SMTC) API，通过 `windows-media-controls` 库 | 网易云音乐、QQ音乐、Spotify 等 | 歌名、艺术家、专辑封面、播放进度 |
| macOS | AppleScript 或 Now Playing 接口 | Music、Spotify 等 | 实时播放信息 |
| 网页端 | Electron DevTools Protocol (CDP)，注入脚本抓取 DOM | 网易云音乐网页版、QQ音乐网页版等 | 网页中的音乐信息 |

#### 6.3.2 历史数据分析

**官方开放平台 API（推荐）：**

| 平台 | 授权方式 | 可获取数据 |
|------|----------|------------|
| 网易云音乐开放平台 | OAuth 2.0 授权登录 | 用户创建/收藏的歌单列表、歌单内详细歌曲信息、听歌排行数据（含播放次数、累计时长）、红心歌曲列表、最近播放记录 |
| QQ音乐开放平台 | 类似 OAuth 授权机制 | 用户音乐行为数据接口 |

**第三方开源方案：**

- 使用 `NeteaseCloudMusicApi` 等开源项目，通过逆向工程获取数据
- 需要用户输入账号密码（注意隐私安全）
- 可获取完整的听歌记录和播放次数

**本地缓存解析：**

- 读取音乐软件的本地缓存文件
- 支持 Electron 架构应用的 Local Storage
- 可提取最近播放列表和本地收藏
- 数据完整性和实时性相对较低

#### 6.3.3 数据应用场景

| 数据类型 | 应用场景示例 |
|----------|-------------|
| 当前播放 | "正在听《晴天》，这首歌很怀旧呢~" |
| 历史歌单 | "发现你收藏了很多周杰伦的歌，看来你是他的忠实粉丝" |
| 播放次数 | "这首歌你已经听了500遍，果然是最爱！" |
| 听歌排行 | "根据你的听歌习惯，推荐这些冷门好歌..." |

### 6.4 视频感知方案

#### 6.4.1 视频内容获取

**B站（哔哩哔哩）：**

- 使用 B站公开 API
- 获取数据：视频标题、分区（如搞笑、科技）、视频标签（Tags）、热门评论、播放量、弹幕数

**抖音/TikTok：**

- 解析分享链接元数据
- 使用开源爬虫工具获取基础描述
- 获取视频话题标签

#### 6.4.2 智能抽帧分析

**触发机制：**

- 窗口标题检测：监测包含"哔哩哔哩"、"抖音"、"播放器"等关键词的窗口
- 仅在视频播放器处于前台时启动分析

**抽帧策略：**

- 每 30-60 秒截取一次屏幕画面
- 使用多模态大模型（Qwen-VL、GPT-4o-mini）进行分析
- Prompt 示例："请根据这张截图和已知视频标题，判断当前剧情是高潮、悲伤还是平淡？"

### 6.5 交互决策模块

#### 6.5.1 情境-反应映射

**规则驱动（低成本）：**

```python
if 歌曲BPM > 120:
    桌宠跟着点头
elif 视频标签包含("恐怖", "惊悚"):
    桌宠做出害怕表情并躲在角落
elif 当前时间 in [23:00, 6:00]:
    桌宠安静陪伴
```

**LLM 总结驱动（高智能）：**

- 每隔 5-10 分钟总结用户行为
- 输入：近期播放的歌曲、观看的视频类型
- 输出：互动建议
- 示例："主人最近看了很多搞笑视频，要不要我讲个笑话？"

#### 6.5.2 防打扰机制

设置"交互冷却时间"（默认 10 分钟），高优先级情境例外处理：

- 检测到生日蛋糕画面
- 用户长时间未操作（主动问候）
- 情绪低落时（播放悲伤音乐）

### 6.6 性能优化

| 优化策略 | 说明 |
|----------|------|
| 异步处理 | 所有感知任务采用异步执行 |
| 资源隔离 | CPU 密集型任务单独进程运行 |
| 缓存机制 | 频繁访问的数据本地缓存 |
| 按需加载 | 非核心功能延迟加载 |

### 6.7 隐私保护

| 保护措施 | 说明 |
|----------|------|
| 明确授权 | 所有数据获取前获得用户授权 |
| 数据脱敏 | 上传云端前进行数据脱敏处理 |
| 本地优先 | 敏感数据尽量在本地处理 |
| 加密存储 | 用户凭证加密存储 |

### 6.8 预期效果

本方案通过"接口直连+低频分析"的优化架构，实现了AI桌宠的多模态感知能力。相比全量分析方案，Token 消耗降低 90% 以上，同时保持了丰富的交互功能。通过历史数据分析，桌宠能够真正理解用户音乐品味，提供更加个性化的陪伴体验。

### 6.1 角色切换组件

```html
<template>
  <div class="character-switcher">
    <div class="current-character">
      <h3>当前角色：{{ currentCharacter.name }}</h3>
    </div>
    <div class="character-list">
      <div v-for="char in availableCharacters" :key="char.id"
           class="character-item" @click="handleSwitchCharacter(char.id)">
        {{ char.name }}
      </div>
    </div>
  </div>

<script>
export default {
  data() {
    return {
      currentCharacter: {},
      availableCharacters: []
    }
  },
  async mounted() {
    await this.loadCharacters()
    await this.loadCurrentCharacter()
  },
  methods: {
    async loadCharacters() {
      const response = await this.$http.get('/api/characters')
      this.availableCharacters = response.data
    },
    async loadCurrentCharacter() {
      const response = await this.$http.get('/api/character/current')
      this.currentCharacter = response.data
    },
    async handleSwitchCharacter(characterId) {
      try {
        const response = await this.$http.post('/api/character/switch', { characterId })
        this.currentCharacter = response.data
        this.$message.success(`已切换至 ${response.data.name}`)
      } catch (error) {
        this.$message.error('切换角色失败')
      }
    },
    openSettings() {
      this.$router.push('/settings')
    }
  }
}
</script>
```

## 八、API接口设计

### 8.1 角色管理API

| 接口 | 方法 | 描述 |
|------|------|------|
| `/api/characters` | GET | 获取所有可用角色列表 |
| `/api/character/current` | GET | 获取当前激活角色信息 |
| `/api/character/switch` | POST | 切换角色 |
| `/api/character/save` | POST | 保存角色设置 |
| `/api/character/chat` | POST | 与当前角色对话 |

### 8.2 API请求/响应示例

**切换角色请求：**


### 9.3 记忆系统运作流程

#### 9.3.1 记忆写入与筛选

**重要性评估机制：**

```python
def evaluate_importance(content):
    """评估对话内容的重要性"""
    importance_rules = [
        (r'生日|纪念日|重要|特别', 0.9),  # 包含重要关键词
        (r'喜欢|讨厌|过敏|害怕', 0.8),    # 个人偏好信息
        (r'天气|温度|冷|热', 0.6),        # 环境相关信息
        (r'你好|再见|谢谢', 0.2),         # 日常问候
    ]

    score = 0.3  # 基础分
    for pattern, weight in importance_rules:
        if re.search(pattern, content):
            score = max(score, weight)
    return score
```

**记忆写入流程：**

1. 对话结束后，评估当前对话的重要性
2. 重要性评分 > 阈值（如 0.5）则进行向量化
3. 将向量存入角色专属的 FAISS 库
4. 同时保存原文和元数据到 SQLite 数据库

#### 9.3.2 记忆检索与召回

**检索算法：**

```python
def retrieve_memories(character_id, query, top_k=3):
    """检索相关记忆"""
    # 1. 将查询内容转化为向量
    query_vector = embedding_model.encode([query])

    # 2. 从角色专属记忆库中检索
    faiss_index = load_faiss_index(f"characters/{character_id}/memory_index.faiss")
    distances, indices = faiss_index.search(query_vector, top_k)

    # 3. 获取对应的记忆内容
    memories = []
    for idx, dist in zip(indices[0], distances[0]):
        memory = get_memory_from_db(character_id, idx)

        # 结合时间衰减和语义相似度计算综合得分
        time_decay = calculate_time_decay(memory['timestamp'])
        score = dist * 0.7 + time_decay * 0.3

        memories.append({
            'content': memory['content'],
            'timestamp': memory['timestamp'],
            'score': score
        })

    return sorted(memories, key=lambda x: x['score'], reverse=True)
```

#### 9.3.3 记忆注入与生成

**动态 Prompt 组装：**

```python
def build_memory_prompt(character_id, user_input):
    """构建包含记忆的提示词"""
    # 检索相关记忆
    relevant_memories = retrieve_memories(character_id, user_input)

    if not relevant_memories:
        return ""

    # 组装记忆提示
    memory_prompt = "[长期记忆]\n"
    for memory in relevant_memories:
        memory_prompt += f"- {memory['content']} (记录于: {memory['timestamp']})\n"

    memory_prompt += "\n[行为指导]\n"
    memory_prompt += "请在回复中自然地引用这些记忆，不要生硬提及，要像真正记得这些事一样。"

    return memory_prompt
```
