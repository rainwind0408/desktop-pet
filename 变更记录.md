### 新增音乐播放记录和分析系统

**变更类型：** 新功能

**变更内容：**

1. **新增 `sensors/music_tracker.py` - 播放记录管理**
   - SQLite 数据库存储播放历史
   - 自动去重（30 秒内同一首歌不重复记录）
   - 支持查询：最近播放、常听歌曲、常听艺术家、播放时段
   - 定期清理旧记录（默认保留 90 天）

2. **新增 `core/music_analyzer.py` - 音乐数据分析**
   - 生成音乐画像（music_profile.json）
   - 统计 7 天/30 天常听 Top 歌曲和艺术家
   - 分析播放时段偏好（早上/下午/晚上/深夜）
   - 基于歌曲名称推断情绪倾向
   - 构建音乐相关系统提示词

3. **修改 `sensors/media_sensor.py` - 扩展平台识别**
   - 新增平台识别：网易云音乐、QQ音乐、Spotify、酷狗、酷我、Apple Music 等
   - 通过 SMTC 来源应用识别平台
   - 新增 `music_platform` 和 `music_album` 状态字段

4. **修改 `core/character_manager.py` - 集成音乐画像**
   - 新增 `set_music_analyzer()` 方法
   - 对话时自动注入音乐偏好提示词

5. **修改 `main.py` - 初始化音乐追踪**
   - 初始化 MusicTracker 和 MusicAnalyzer
   - 媒体感知回调自动记录播放

**数据库结构（music_history.db）：**
```sql
CREATE TABLE play_history (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    title       TEXT NOT NULL,
    artist      TEXT DEFAULT '',
    album       TEXT DEFAULT '',
    platform    TEXT DEFAULT '',
    play_time   DATETIME NOT NULL,
    duration    INTEGER DEFAULT 0,
    source      TEXT DEFAULT 'smtc'
);
```

**音乐画像示例（music_profile.json）：**
```json
{
  "stats": {"total_plays": 1256, "unique_songs": 342},
  "top_songs_7d": [{"title": "晴天", "artist": "周杰伦", "plays": 12}],
  "top_artists_30d": [{"artist": "周杰伦", "plays": 120}],
  "listening_pattern": {"morning": 15, "afternoon": 25, "evening": 40, "night": 20}
}
```

- 酷我音乐 ✓

### 增强天气模块：多数据源支持和 IP 定位

**变更类型：** 功能增强

**变更内容：**

1. **新增 Open-Meteo 天气数据源**
   - 作为 wttr.in 的备用数据源
   - 免费、无需 API Key、高精度
   - 使用 WMO 天气代码标准

2. **新增 IP 自动定位**
   - 使用 ip-api.com 免费服务
   - 自动识别用户所在城市
   - 定位失败时默认使用北京坐标

3. **新增天气描述翻译**
   - WMO 天气代码 → 中文描述（覆盖 0-99 所有代码）
   - 英文天气描述 → 中文翻译回退机制

4. **完善风力描述**
   - 6 级风力描述：无风/微风/轻风/中风/大风/狂风

5. **优化天气获取逻辑**
   - 多数据源自动切换：wttr.in → Open-Meteo → 备用缓存
   - 优化 wttr.in 中文天气描述提取

**数据源优先级：**
```
1. wttr.in（主）    - 免费，全球可用，支持中文
2. Open-Meteo（备） - 免费，高精度，WMO 标准
3. 本地缓存（兜底） - 网络失败时使用
```

- `sensors/environment_sensor.py` - 重写天气获取模块

### 重构 LLM 模块配置管理

**变更类型：** 功能优化

**变更内容：**

1. **新建 `llm/` 模块目录**
   - `llm/__init__.py` - 模块入口
   - `llm/provider.py` - LLM 提供者类（BaseLLMProvider, OpenAICompatibleProvider, AnthropicProvider）
   - `llm/factory.py` - LLM 工厂和配置管理函数

2. **每个提供商独立配置文件**
   - `llm/config/deepseek.json`
   - `llm/config/openai.json`
   - `llm/config/qwen.json`
   - `llm/config/zhipu.json`
   - `llm/config/moonshot.json`
   - `llm/config/anthropic.json`
   - `llm/config/baidu.json`
   - `llm/config/minimax.json`
   - `llm/config/stepfun.json`
   - `llm/config/doubao.json`
   - `llm/config/ollama.json`
   - `llm/config/custom.json`

3. **修改文件**
   - `main.py` - 使用新的 `llm` 模块导入
   - `api_server.py` - 更新配置读写逻辑，新增 `/api/config/llm/provider` 接口
   - `pet_window.py` - 重写 `SettingsDialog`，切换提供商时自动保存/加载配置
   - `.gitignore` - 添加 `llm/config/*.json` 保护 API Key

4. **删除文件**
   - `llm_provider.py` - 已移到 `llm/` 模块

**解决问题：**

- 之前 `llm_config.json` 只保存全局配置，切换模型时会覆盖之前的 API Key，导致每次更换模型都需要重新填写
- 现在每个提供商的配置（包括 API Key）独立保存在 `llm/config/{provider}.json`，切换时自动加载，无需重复填写

**使用说明：**

- 切换提供商时，系统自动保存当前配置到对应文件
- 加载新提供商时，自动读取对应的配置文件
- API Key 等敏感信息已加入 `.gitignore`，不会被提交到版本控制

### 优化记忆系统性能

**变更类型：** 性能优化

**优化内容：**

| 优化项 | 原实现 | 新实现 | 收益 |
|--------|--------|--------|------|
| SQLite 连接 | 每次操作开关连接 | 连接池 + 长连接 | 减少 I/O 开销 |
| 摘要生成 | 每次写入立即重建 | 延迟 10 秒批量更新 | 减少写入延迟 |
| FAISS 索引 | IndexFlatIP 暴力搜索 | IndexIVFFlat (数据>1000) | 提升检索速度 |
| 缓存策略 | 固定 5 分钟 TTL | 动态 5-30 分钟 | 提高命中率 |
| 重要性评估 | 纯关键词匹配 | 可选 LLM 辅助 | 提升准确性 |
| 解密缓存 | 每次读取都解密 | 缓存解密结果 | 减少重复计算 |

**新增功能：**
- `SQLiteConnectionPool` - SQLite 连接池，支持空闲清理
- `MemorySystem.cleanup()` - 程序退出时清理资源
- 自动索引升级：数据量超过 1000 时自动切换到 IVFFlat

**修改文件：**
- `core/memory_system.py` - 重构记忆系统，整合所有优化
- `main.py` - 添加 atexit 注册 cleanup 清理资源
