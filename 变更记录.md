## 2026-05-19

### 修复渲染引擎版本兼容性及多项渲染问题

**变更类型：** Bug 修复

**变更内容：**

1. **P0: 修复 GLTFLoader 404 导致 VRM 无法渲染**
   - 问题：Three.js 0.160.0 已移除 `examples/js/` 目录，GLTFLoader.js 和 OrbitControls.js 返回 404
   - 根因：Three.js ≥ 0.154.0 仅保留 ES Module 版本，与当前 `<script>` 标签加载方式不兼容
   - 修复：降级到 Three.js 0.146.0 + three-vrm 1.0.6（最后兼容的全局变量版本组合）
   - 版本验证：GLTFLoader.js (112KB)、OrbitControls.js (26KB) 均返回 200

2. **P1: 实现 Live2D 动作支持**
   - 问题：`playMotion()` 中 Live2D 部分是空注释，点击无反应
   - 修复：调用 `pixi-live2d-display` 的 `_live2dModel.motion(group)` API

3. **P2: 修复内存泄漏**
   - 问题：`cleanup()` 未销毁 Three.js 的 Geometry/Material/Texture
   - 修复：遍历 scene 子对象，逐一 dispose geometry、material、texture
   - 新增 VRM 模型的 dispose 调用

4. **P2: 降低重复加载风险**
   - 问题：`window.onload` 定时器 500ms 太短，Python 注入可能延迟
   - 修复：超时时间从 500ms 增加到 2000ms

**版本变更：**

| 依赖 | 旧版本 | 新版本 | 原因 |
|------|--------|--------|------|
| Three.js | 0.160.0 | 0.146.0 | 保留 examples/js/ 全局变量版本 |
| three-vrm | 2.0.6 | 1.0.6 | 兼容 three@0.146.0 |

- 之前 `llm_config.json` 只保存全局配置，切换模型时会覆盖之前的 API Key，导致每次更换模型都需要重新填写

**使用说明：**

- 切换提供商时，系统自动保存当前配置到对应文件
- 加载新提供商时，自动读取对应的配置文件
- API Key 等敏感信息已加入 `.gitignore`，不会被提交到版本控制

### 优化记忆系统性能

**变更类型：** 性能优化

**优化内容：**

| 优化项 | 原实现 | 新实现 | 收益 |
|--------|--------|--------|------|
| SQLite 连接 | 每次操作开关连接 | 连接池 + 长连接 | 减少 I/O 开销 |
| 摘要生成 | 每次写入立即重建 | 延迟 10 秒批量更新 | 减少写入延迟 |
| FAISS 索引 | IndexFlatIP 暴力搜索 | IndexIVFFlat (数据>1000) | 提升检索速度 |
| 缓存策略 | 固定 5 分钟 TTL | 动态 5-30 分钟 | 提高命中率 |
| 重要性评估 | 纯关键词匹配 | 可选 LLM 辅助 | 提升准确性 |
| 解密缓存 | 每次读取都解密 | 缓存解密结果 | 减少重复计算 |

**新增功能：**
- `SQLiteConnectionPool` - SQLite 连接池，支持空闲清理
- `MemorySystem.cleanup()` - 程序退出时清理资源
- 自动索引升级：数据量超过 1000 时自动切换到 IVFFlat

**修改文件：**
- `core/memory_system.py` - 重构记忆系统，整合所有优化
- `main.py` - 添加 atexit 注册 cleanup 清理资源
